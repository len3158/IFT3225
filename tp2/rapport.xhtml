L'ensemble des elements coeurs du jeu utilisent l'aspect POO de JavaScript et Jquery. Le jeu comprends donc les attributs locaux propres au jeu agissant comme des parametres ainsi que trois constructeurs principaux :

  • Un constructeur du jeu
  • Un constructeur pour les elements du DOM
  • Un constructeur pour instantier tous les event listeners necessaires

La syntaxe de notre code se conforme aux regles de Jquery (comme par exemple le nom des variables locales commencent par le caractere "$". La grille de jeu est imbriquee dans des elements "div", et genere un tableau selon les dimensions specifiees. La principale difficulte etait de diviser correctement l'image, sans perdre ses proprotions. Nous sommes passes par plusieurs approches (canvas, elements html ...) avant de trouver ce que nous estimons etre l'approche la plus ergonomique et facile a implementer et comprendre.



Le code se divise donc en trois elements principaux :

  1. Declaration des variables
  2. Instanciation des elements de la page
  3. Definition des fonctions generatrices des case en recuperant l'image passee en parametre
  4. Fonction logiques gerant les deplacements, l'echange de case et verifiant la completion de la grille
  5. Et enfin l'instantiation de la grille de jeu
